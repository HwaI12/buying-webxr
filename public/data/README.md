# 色配置パターン評価実験 データ構造

## 概要

本ディレクトリには、WebVR環境での色配置パターン評価実験に必要なデータファイルが格納されています。

### 評価実験の目的

**色配置パターンの違いが、被験者の主観的な購買意欲や印象評価に差を生むことを示し、その差を合理的に説明すること**

本実験は「画像ベースアンケート」として設計されています。WebVRシステムで各配置パターンを描画し、その**スクリーンショット画像**を撮影し、画像と質問を組み合わせたアンケート形式で評価データを収集します。視線追跡や行動ログは取得しません。

卒業論文で求められるのは以下の3点です：

1. **パターン間で評価に差が出ること**（統計的有意差）
2. **その差が「色配置の違い」によると説明できること**
3. **結果が既存知見と整合していること**（または矛盾の理由を説明できること）

## 実験設計

- **実験デザイン**: 被験者内計画（Within-subject design）
- **独立変数**: 色配置パターン（10条件）
- **従属変数**: 購買意欲、魅力度、見やすさ（各7件法リッカート尺度）
- **統制変数**: 照明、商品形状、数量、視点位置
- **データ収集方法**: WebVR画面のスクリーンショット画像 + アンケート（Google Forms等）
- **視線追跡・行動ログ**: なし

## 各ファイルの説明

### 1. environment-config.json

環境設定ファイル。全パターンで共通の実験環境を定義。

- **照明**: 昼白色5000K、強度1.0
- **ブース**: 白い壁、明るいグレーの床
- **視点**: 目の高さ1.6m、距離2.2m
- **グリッド**: 2ショーケース × 4段 × 3列 = 計24個

### 2. product-catalog.json

商品カタログファイル。ケーキの色定義と色彩理論の情報。

- **暖色**: red, orange, pink, brown（食欲喚起）
- **寒色**: lime（爽やかさ、フレッシュさ）
- **中性色**: white（基準色）

### 3. experiment-config.json

実験設定ファイル。実験フロー、質問項目、データエクスポート設定。

- **実験フロー**: 同意 → 基本情報 → 説明 → 練習試行 → 本試行 → 事後質問
- **質問項目**:
  - 購買意欲（7件法）
  - 魅力度（7件法）
  - 見やすさ（7件法）
  - 自由記述（任意）
- **統計解析**: 対応のある分散分析、多重比較、効果量

### 4. patterns-index.json

パターン一覧とメタデータ。実験設計上の分類と比較軸を定義。

**比較軸**:
- **色の多様性**: 単色 vs 多色
- **色温度**: 暖色 vs 寒色 vs 混合
- **対称性**: コピー vs 鏡像 vs 非対称
- **配置規則性**: グラデーション vs 構造的 vs ランダム

### 5. patterns/ ディレクトリ

個別パターンファイル（計10個）。各ファイルには以下が含まれます：

- パターンID、名前、説明
- 研究目的と期待される結果
- 配置要因（色多様性、色温度、対称性など）
- 24個のお皿への具体的な配置

## 10個のパターン概要

### 基準条件（Baseline）

1. **pattern-01-monochrome-white**: 単色（白）
   - 対照条件、刺激なし

2. **pattern-02-monochrome-red**: 単色（赤）
   - 暖色の効果検証

### 色調和（Harmony）

3. **pattern-03-complementary-accent**: 補色アクセント
   - 補色対比の効果

### 色温度（Temperature）

5. **pattern-05-cool-dominant-warm-accent**: 寒色優位＋暖色アクセント
   - 寒色主体の配置

8. **pattern-08-warm-dominant**: 暖色優位
   - 暖色主体の配置（食欲喚起仮説の検証）

### グラデーション（Gradient）

4. **pattern-04-warm-analogous-gradient**: 暖色類似＋グラデーション
   - 対角グラデーション

10. **pattern-10-vertical-gradient**: 垂直グラデーション
    - 上から下へのグラデーション

### 対称性（Symmetry）

6. **pattern-06-symmetric-copy**: 同位置対称（完全一致）
   - 左右完全対称

9. **pattern-09-mirror-symmetric**: 鏡像対称
   - 左右鏡像反転

7. **pattern-07-random-asymmetric**: ランダム配置（非対称）
   - 非対称、ランダム配置

## 実験における主要な仮説

1. **仮説1（色の多様性）**: 単色より多色配置の方が購買意欲が高い
   - 比較: pattern-01, 02 vs pattern-03, 04, 06, 08

2. **仮説2（色温度）**: 暖色優位配置は寒色優位配置より購買意欲が高い
   - 比較: pattern-02, 04, 08 vs pattern-05

3. **仮説3（対称性）**: 対称配置は非対称配置より見やすさ評価が高い
   - 比較: pattern-06, 09 vs pattern-07

4. **仮説4（配置規則性）**: グラデーション配置はランダム配置より魅力度が高い
   - 比較: pattern-04, 10 vs pattern-07

## 統計解析の推奨事項

- **被験者数**: 20-30名（被験者内計画）
- **検定方法**: 対応のある一元配置分散分析
- **多重比較**: Bonferroni補正またはHolm法
- **効果量**: 偏イータ二乗
- **有意水準**: α = 0.05

## 論文での記述例

### 実験目的の記述

> 本評価実験は、複数の色配置パターンを提示し、被験者の主観的評価（購買意欲・魅力度・見やすさ）を基に、色配置の違いが購買意欲に与える影響を比較・検討することを目的とする。

### 実験方法の記述

> 本研究では、WebVR環境を用いて10種類の色配置パターンを被験者に提示し、各パターンに対する購買意欲、魅力度、見やすさを7件法リッカート尺度で評価した。実験デザインは被験者内計画とし、パターンの提示順序はカウンターバランスのためランダム化した。環境条件（照明、商品形状、数量、視点位置）は全パターンで統一し、色配置のみを独立変数として操作した。

### 結果の記述例

> 対応のある一元配置分散分析の結果、色配置パターンは購買意欲に有意な影響を与えることが示された（F(9, 189) = 12.34, p < .001, η²p = .37）。事後検定（Bonferroni補正）の結果、暖色優位パターン（M = 5.8, SD = 1.0）は単色（白）パターン（M = 2.5, SD = 1.2）と比較して有意に高い購買意欲評価を得た（p < .001）。この結果は、暖色が食欲・購買意欲を高めるという既存の色彩心理研究（Singh, 2006; Labrecque & Milne, 2012）と整合する。

## 使用方法

### 実験実施の流れ

#### ステップ1: WebVRシステムでパターン画像を生成

1. **環境設定の読み込み**: `environment-config.json`
   - 照明、ブース、グリッド配置などの基本設定
2. **商品カタログの読み込み**: `product-catalog.json`
   - ケーキの色定義と色彩理論の情報
3. **パターン一覧の取得**: `patterns-index.json`
   - 実験で使用する10個のパターンの一覧と分類
4. **個別パターンの読み込みと描画**: `patterns/pattern-XX-*.json`
   - 各パターンの具体的な配置データを読み込み、WebVRで描画
5. **スクリーンショット撮影**: 各パターンのVR画面をスクリーンショットとして保存

#### ステップ2: アンケートの作成と実施

1. **アンケートツールの選択**: Google Forms、Microsoft Forms、Qualtrics等
2. **実験設定の参照**: `experiment-config.json`
   - 実験フロー、質問項目、データエクスポート設定を確認
3. **アンケート作成**:
   - 撮影したパターン画像を各質問に挿入
   - 7件法リッカート尺度の質問を設定（購買意欲、魅力度、見やすさ）
4. **アンケート実施**: 被験者にパターン画像を見せながら評価を収集

### データの流れ

```
【準備段階】
WebVRシステム起動
  ↓
10個のパターンを順次表示
  ↓
各パターンのスクリーンショット撮影
  ↓
アンケート作成（画像を挿入）

【実験段階】
被験者情報入力（年齢、性別、色覚）
  ↓
実験説明・練習試行（任意）
  ↓
本試行（10パターン画像をランダム順序で提示）
  │
  ├─ パターンA画像を表示 → 質問に回答（購買意欲、魅力度、見やすさ）
  ├─ パターンB画像を表示 → 質問に回答
  ├─ ...
  └─ パターンJ画像を表示 → 質問に回答
  ↓
事後質問（最良パターン、最劣後パターン、ランキング）
  ↓
データのエクスポート（CSV形式）
  ↓
統計解析（対応のある分散分析、多重比較）
```

## 既存知見との整合性

- **暖色**: 食欲・購買意欲が高まる（既存研究と整合）
- **寒色**: 爽やかさ・フレッシュさ（刺激は弱い）
- **対称配置**: 秩序感・見やすさ
- **ランダム配置**: 情報過多で評価低下の可能性

## 注意事項

### スクリーンショット撮影時

- **統一された視点**: 全パターンで同じカメラ位置・角度からスクリーンショットを撮影すること
- **環境条件の統一**: 照明、ブース、グリッド配置などを全パターンで統一すること（`environment-config.json`の設定を維持）
- **画像解像度**: 全パターンで同じ解像度のスクリーンショットを撮影すること
- **ファイル命名**: パターンIDに対応したファイル名で保存すること（例: `pattern-01-monochrome-white.png`）

### アンケート実施時

- **パターン順序のランダム化**: 被験者ごとにパターン提示順序をランダム化し、カウンターバランスを取ること
- **画像の品質**: アンケートツールで画像が鮮明に表示されることを確認すること
- **被験者の疲労**: 10パターン×3質問=30項目あるため、適切な休憩や説明を設けること
- **倫理的配慮**: 同意取得と途中終了の権利を保証すること

### データ分析時

- 外れ値の確認と処理方針を明記すること
- 球面性の仮定（Mauchlyの検定）を確認すること
- 違反している場合はGreenhouse-Geisser補正を適用
- 多重比較の際は必ず補正を行うこと
- 効果量を必ず報告すること

### 論文執筆時

- 統計的有意差だけでなく、実質的な意味を考察すること
- 結果を既存知見と比較し、整合性・矛盾を議論すること
- 限界（VR環境特有の制約、被験者数など）を明記すること
- 再現可能性のため、実験手続きを詳細に記述すること
